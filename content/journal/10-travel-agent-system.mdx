I started by trying to stop refreshing flight prices.

That was the whole ambition. I was planning a trip, had a few destinations in mind, and kept checking prices manually like everyone does. Opening the same tabs, comparing the same numbers, watching prices move and never knowing when to pull the trigger. I wanted that problem to go away.

So I opened Claude and described what I wanted in plain English. Track flights across these destinations. Watch this time range. Tell me when something looks right. No code. No configuration. I described a job the way I would describe it to a person and the agent understood it.

It worked immediately. I sat there waiting for something to break and nothing broke. Real flights, real prices, real windows. I added a hotel agent the same afternoon using the same approach. Then a third that pulled both outputs together and gave me a total cost breakdown per option. Flight plus hotel, per destination, per time window, updated automatically.

In one afternoon I went from manually refreshing browser tabs to a three-agent system running searches across multiple destinations simultaneously, aggregating costs into a single view, and running without me touching anything.

I am not a developer. I had never built an agent before. I did not study this. I described what I wanted in plain English and it worked. That afternoon changed how I think about what is possible.
